# 复盘溯源（事后审核模块）

> 功能定位：直播结束后或封禁发生后，对整场直播进行回溯分析，输出合规复盘报告
> 核心价值：从违规中学习，持续优化规则库和模型

---

## 使用方式

```
/视频号审核:复盘溯源 <复盘内容> [复盘类型：常规/封禁]
```

**复盘内容支持：**
- 直播录屏转文字稿
- 平台封禁通知/违规记录截图描述
- 人工记录的违规时间点和原话
- AI预审报告 vs 实际情况对比

**复盘类型：**
- `常规`：每场直播结束后的例行复盘
- `封禁`：发生封禁后的深度复盘（必做）

---

## 常规复盘流程

### STEP 1：数据收集

| 数据项 | 来源 | 格式 |
|--------|------|------|
| 全场话术文字稿 | 录屏转写 | 纯文本 |
|本场AI预警记录 | 风险预警模块导出 | JSON |
| 中控人工记录 | 中控备忘录 | 文本/表格 |
| 直播数据（在线峰值/停留时长/互动率） | 后台数据 | 数值 |
| 平台处罚记录（如有） | 通知截图 | 图片/文本 |

### STEP 2：回溯分析

将全场话术重新跑一遍完整审核流程：

```
全场话术 → 话术预审(全量) → 合规自检(完整模式) → 风险预警(全量)
    ↓
对比本场实时预警记录：
  - 预警到了但没改的 → 执行力问题
  - 没预警到但实际违规的 → 漏检问题（重点分析）
  - 预警了也改了但还是被封的 → 规则覆盖不全问题
    ↓
标记所有"应检未检"项
```

### STEP 3：漏检根因分析

对每个漏检项进行5Whys分析：

| 漏检类型 | 可能原因 | 改进措施 |
|---------|---------|---------|
| 新型违规表达 | 规则库未收录 | 补充规则库+关键词库 |
| 语义链路变体 | 逻辑链模式不够完善 | 扩展Chain模式 |
| 上下文误判 | 敏感词安全/危险语境判断错误 | 调整上下文判定阈值 |
| 组合违规 | 单个词安全但组合起来违规 | 增加组合检测规则 |
| 平台新规 | 平台更新了规则 | 同步更新规则库 |

### STEP 4：输出复盘报告

## 封禁复盘流程（深度版）

### STEP 0：紧急信息收集

封禁发生后**1小时内**完成：

| 信息项 | 说明 |
|--------|------|
| 封禁时间 | 精确到分钟 |
| 封禁类型 | 弹窗提醒/减少推荐/中断直播/禁播/封禁账号 |
| 封禁原因（平台告知） | 平台给出的违规原因描述 |
| 封禁前最后5分钟话术 | 封禁前说的最后几段话 |
| 当场是否收到AI预警 | 是/否，预警等级 |
| 主播是否执行了整改 | 是/否 |

### STEP 1：深度归因

```
平台封禁原因
    ↓
映射到内部规则编号（可能一对多）
    ↓
定位到具体话术片段（精确到句）
    ↓
分析为什么AI没有提前拦截：
  a) 规则库缺失 → 补规则
  b) 语义理解偏差 → 调模型
  c) 预警了但没执行 → 流程/培训问题
  d) 新型规避手法 → 更新检测策略
    ↓
生成"封禁案例"录入案例库
```

### STEP 2：规则库更新

每次封禁复盘后必须执行的更新操作：

```
┌─ 新违规表达 ──→ 加入关键词库（关键词库.md）
├─ 新违规逻辑链 ──→ 加入违规规则库（违规规则库·五）
├─ 新违规案例 ──→ 加入违规案例库（违规案例库.md）
└─ 规则/词调整 ──→ 更新版本号 + 记录变更日志
```

## 规则迭代机制

### 周迭代（轻量）
- 统计本周新增违规类型TOP3
- 更新关键词库（新增词+调整上下文判定）
- 微调语义链路检测灵敏度
- 输出：《周规则变更日志》

### 月迭代（重量）
- 全量回顾本月封禁案例
- 评估AI识别准确率（= 正确预警数 / 总预警数）
- 评估漏检率（= 漏检违规数 / 总实际违规数）
- 评估误判率（= 误判预警数 / 总预警数）
- 目标：准确率≥98%，误判率≤2%
- 输出：《月度合规运营报告》
