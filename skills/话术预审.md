# 话术预审（事前审核核心模块）

> 功能定位：直播��对脚本/口播文案进行全量合规预审，输出违规标记+修改建议
> 输入：直播脚本、口播文案、逐字稿文本
> 输出：结构化审核报告（违规清单+风险等级+修改建议）

---

## 使用方式

```
/视频号审核:话术预审 <话术文本或文件路径> [业务线：有声投放/IP自然流/IP投流] [严格度：标准/严格]
```

**三条业务线说明：**

| 业务线 | 适用场景 | 审核严格度 | 核心风险 |
|--------|---------|-----------|---------|
| **有声投放** | 喜播教育/喜马拉雅生态下的视频号投流直播 | ★★★★★ 最严（双审：内容+广告） | 喜马残留+利益诱导+招聘嫌疑 |
| **IP自然流** | 张震工作室自然流直播（连麦+关注+加助教） | ★★★☆☆ 相对宽松（仅内容审） | 引流方式+连麦合规+低质内容 |
| **IP投流** | 张震工作室视频号投流直播（卡片+助教转化） | ★★★★☆ 严格（双审：内容+广告） | 收益隐喻+虚假紧迫感+广告法违规 |

不指定业务线时默认**"有声投放"**（最严模式，向下兼容）；不指定严格度时默认"标准"。

**选择业务线后AI的行为差异：**

| 处理行为 | 有声投放 | IP自然流 | IP投流 |
|---------|---------|---------|--------|
| 加载规则 | 全量 + AY(有声)10条专项 | 全量 + IPN(自然流)8条专项 | 全量 + IPI(投流)10条专项 |
| "喜马"词权重 | ×5（零容忍） | ×3 | ×3 |
| "鉴定"作为福利 | 🔴 一定报警 | ✅ 连麦场景放行 | 🔴 报警 |
| "收获/获得/口袋" | 🔴 高敏 | 🟡 中敏 | 🔴×5 超高敏（暗语重灾区） |
| 白名单 | 腾讯广告白名单 | 自然流宽松白名单 | 腾讯广告白名单 |

详见 `rules/业务线差异化规则.md`

## 审核流程（6步法）

### STEP 0：确定业务线（新增）

根据用户指定的业务线参数，加载对应规则集：

```
指定"有声投放" → 加载 rules/业务线差异化规则.md 的 2.1节 + AY-01~AY-10
指定"IP自然流"   → 加载 rules/业务线差异化规则.md 的 2.2节 + IPN-01~IPN-08
指定"IP投流"     → 加载 rules/业务线差异化规则.md 的 2.3节 + IPI-01~IPI-10
未指定           → 默认"有声投放"模式
```

### STEP 1：文本预处理

- **输入接收**：支持纯文本粘贴、文件路径（.txt/.docx/.md）
- **分段切分**：按句子/语义段切分为审核单元
- **元信息提取**：识别话术环节（开场钩子/干货教学/价值塑造/逼单引导/收口过渡）
- **预处理输出**：总句数、各环节数量、预估时长

### STEP 2：关键词精确扫描

加载 `rules/关键词库.md`，执行三级检测：

**Level 1 - 精确匹配：**
- 逐句比对绝对化用语词表
- 逐句比对收益赚钱词表
- 逐句比对品牌违禁词表
- 逐句比对招聘引流词表
- 逐句比对效果承诺词表

**Level 2 - 模糊匹配：**
- 包含匹配（关键词是句子子串）
- 近义词替换检测（搞钱→赚钱，某信→微信）
- 变体/绕过检测（拼音缩写、谐音、emoji替换）

**Level 3 - 上下文敏感判断：**
- 加载上下文敏感词表，判断安全/危险场景
- 标注"单独安全但语境危险"的词汇

### STEP 3：语义逻辑链分析

这是2026年视频号审核的核心——**不只看单个词，看整段话传递的含义**。

#### 3.1 高风险逻辑链检测

| 链路编号 | 违规逻辑链 | 有声投放灵敏度 | IP自然流灵敏度 | IP投流灵敏度 |
|---------|-----------|--------------|-------------|------------|
| Chain-01 | 学习课程→创造价值→获得回报→改善生活 | ★★★★★ | ★★★☆☆ | ★★★★★ |
| Chain-02 | 学习课程→获得角色→获得录制机会→发展机会 | ★★★★★ | ★★☆☆☆ | ★★★★☆ |
| Chain-03 | 学习课程→获得优势→改变未来 | ★★★★★ | ★★★☆☆ | ★★★★★ |
| Chain-04 | 完成指定动作→获得额外福利 | ★★★★☆ | ★★☆☆☆ | ★★★★★ |
| Chain-05 | 低投入→低门槛→易获得结果 | ★★★★☆ | ★★★☆☆ | ★★★★☆ |

#### 3.2 检测算法

```
对每段话术：
1. 提取谓语动词和核心名词
2. 构建语义依赖图（主语→谓语→宾语→补语）
3. 匹配高风险逻辑链模式（按当前业务线的灵敏度）
4. 计算链路完整度（完整命中 > 部分命中 > 单点命中）
5. 输出命中的链路编号 + 触发片段 + 风险等级
```

### STEP 4：业务线专项规则扫描

根据所选业务线，执行专属规则检查：

**有声投放额外检查：**
- [ ] 喜马品牌残留扫描（"喜马拉雅/喜马/官方"任何形式）
- [ ] 声音鉴定是否作为报名交换条件
- [ ] 是否有兼职/接单/录书赚钱暗示
- [ ] 助教带练是否有过度承诺

**IP自然流额外检查：**
- [ ] 连麦内容是否混入销售转化
- [ ] 引流路径是否合规（点头像→主页→扫码）
- [ ] 无人连麦时是否有低质内容风险
- [ ] 收益表达是否超出"兴趣爱好"边界

**IP投流额外检查：**
- [ ] 收益暗语扫描（"口袋/满满/获得/收获/获得感"）
- [ ] 紧迫感是否真实可验证
- [ ] 是否有"接书路径"直接暗示
- [ ] 播放量-收益关联表达
- [ ] 卡片引导频率是否过高

### STEP 5：规则库交叉验证

将STEP2~4的结果与 `rules/违规规则库.md` + `rules/业务线差异化规则.md` 交叉匹配：

| 命中类型 | 处理方式 |
|---------|---------|
| 关键词 + 通用规则 + 业务线规则同时命中 | 取最高风险等级 |
| 仅关键词命中无规则对应 | 标注"疑似违规，需人工复核" |
| 仅规则逻辑链命中（无明确违禁词） | 标注"隐性违规"，给出高风险警告 |
| 关键词在白名单中 | 标记为合规，跳过 |
| 当前业务线允许但其他线禁止的 | 标注"[XX线允许]" |

### STEP 6：生成审核报告

## 与其他模块联动

- 预审通过后上线 → `skills/合规自检.md` 做二次确认（自动带入业务线参数）
- 预审发现问题 → `rules/关键词库.md` 查询详细违规定义
- 预审发现问题 → `rules/业务线差异化规则.md` 查看业务线专属规则
- 需要生成合规版本 → 复用 `skill-涂涂/获客直播skill包/直播脚本.md` 的生成能力
- 直播后复盘 → `skills/复盘溯源.md` 对比预审结果与实际违规情况
