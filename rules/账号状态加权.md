# 账号状态加权机制

> 来源：逻辑对齐稿 + 历史封禁数据经验
> 核心原则：同样一句话，在不同账号状态下风险完全不同

---

## 为什么需要账号状态加权

**关键发现（来自历史封禁样本）：**

> "同样一句'让你的荷包更加丰满'，正常账号可能只是收到弹窗提醒，但刚禁播恢复的账号直接被中断直播。"

平台不是完全按固定规则判罚的，**账号健康度是重要变量**：
- 近期有违规记录 → 平台盯得更紧 → 同样话术处罚更重
- 刚禁播恢复 → 处罚升级最明显
- 正常账号 → 有一定容错空间

因此审核系统不能只看"这句话违不违规"，还要看**"这个账号现在能不能承受这句话"**。

---

## 四级账号状态定义

| 状态等级 | 定义 | 时间窗口 | 特征 |
|---------|------|---------|------|
| 🟢 **正常** | 无近30天内无任何违规记录 | 近30天清白 | 正常容错空间 |
| 🟡 **观察期** | 近30天内有轻度违规（弹窗提醒/减少推荐） | 30天内 | 容错空间缩小，中度风险可能被加重 |
| 🟠 **高危期** | 近14天内有中度违规（中断直播/禁播<24h）或累计2次以上轻度违规 | 14天内 | 容错空间极小，低危可能被升为中危 |
| 🔴 **恢复期** | 近7天内解禁恢复 / 或近30天内有过封禁(≥24h) | 7-30天内 | **零容错，所有风险自动升一级** |

### 状态判定规则

```
输入：账号ID + 当前时间
    ↓
查询该账号最近违规记录
    ↓
┌─ 近7天内有封禁(≥24h) ─────────────→ 🔴 恢复期
│
├─ 近14天内有中断/禁播(<24h) ────────→ 🟠 高危期
│
├─ 近30天内有弹窗/减推荐 ────────────→ 🟡 观察期
│
└─ 近30天无任何记录 ─────────────────→ 🟢 正常
    ↓
输出：当前账号状态等级
```

## 加权规则

### 基础规则：风险等级上调

| 原始风险等级 | 🟢正常 | 🟡观察期 | 🟠高危期 | 🔴恢复期 |
|-------------|--------|---------|---------|---------|
| 线线 | 线线 | **↑ 高危** | **↑🔴 红线** | **↑🔴 红线** |
| 高危 | 高危 | **↑🔴 红线** | **↑🔴 线线** | **↑🔴 红线** |
| 中危 | 中危 | **↑ 高危** | **↑ 高危** | **↑🔴 红线** |
| 低危 | 低危 | ↑ 中危 | **↑ 中危** | **↑ 高危** |

### 特殊规则：特定表达 × 账号状态的致命组合

以下组合**无论原始风险等级如何，直接判定为红线**：

| 致命组合 | 触发条件 | 原因 |
|---------|---------|------|
| 收益表达 + 🔴恢复期 | 任何"赚钱/收入/收获/获得"相关表达 | 恢复期零容忍，平台重点监控 |
| 接单暗示 + 🟠高危期 | "接单/录制/角色/机会" | 高危期+兼职暗示=必杀 |
| 喜马残留 + 任何非🟢状态 | "喜马/官方"任何形式 | 非正常状态下的品牌违规从重处理 |
| 利益交换 + 🟡观察期 | "扣已加/报名送XX" | 观察期的诱导行为被放大审查 |

### 反向规则：正常账号的适度放宽

| 场景 | 🟢正常账号特殊处理 |
|------|-------------------|
| 低危违规 | 仅记录不拦截，下轮优化即可 |
| 单个中危（非组合） | 可开播但必须标注，直播中注意 |
| 已验证安全的话术 | 标记为"已验证安全"，后续免检 |

## 在审核流程中的接入位置

### 事前预审时

```
输入话术 → 关键词扫描 → 语义分析 → 得到基础风险等级
                                              ↓
                                    输入账号状态
                                              ↓
                              ┌─ 账号状态加权 ──┐
                              │ 应用上调规则    │
                              │ 检查致命组合    │
                              └───────┬────────┘
                                      ↓
                              输出最终风险等级 + 加权说明
```

### 封禁复盘时

```
输入转写文本 + 平台处罚信息 + 账号状态
    ↓
时间线扫描 → 定位疑似触发段
    ↓
回看当时账号状态 → 判断是否因状态导致处罚加重
    ↓
输出："这段话在正常账号可能是X级，但在[当时状态]被判定为Y级"
```

---

## 输出格式中的展示

在审核报告和预警中加入字段：

```text
账号状态：🟢 正常 / 🟡 观察期 / 🟠 高危期 / 🔴 恢复期
加权说明：原始风险[高危] + 账号状态[观察期] = 最终[红线]
建议：该账号当前状态下，建议将此表述替换为安全版本
```
